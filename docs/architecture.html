Factory Inventory Management — System Architecture

Full-stack demo: Vue 3 frontend, FastAPI backend, in-memory JSON mock data

Tech Stack

Frontend

Vue 3 + Vite

  • vue ^3.4 — Composition API
  • vue-router ^4.3 — client-side routing
  • axios ^1.6 — HTTP client
  • vite ^5.2 — dev server / bundler
  • Served on localhost:3000
Backend

Python + FastAPI

  • fastapi >=0.110
  • uvicorn >=0.24 — ASGI server
  • pydantic >=2.5 — validation
  • Managed with uv
  • Served on localhost:8001
Data

In-memory mock data

  • No database
  • JSON files in server/data/
  • Loaded once at startup via mock_data.py
  • Changes reset on server restart

Data Flow

1

User adjusts a filter

Time Period, Warehouse, Category, or Order Status in FilterBar.vue, backed by the useFilters composable.

2

View calls api.js

The active view (e.g. Dashboard.vue) calls a method like api.getDashboardSummary(filters), which serializes filters into query params.

3

axios request to FastAPI

HTTP GET hits localhost:8001/api/...; CORS is open for the dev frontend origin.

4

In-memory filtering

The endpoint filters the relevant in-memory list (skipping any filter set to "all") and returns matching records.

5

Pydantic validation

Response is validated and serialized against a typed response_model before being sent back as JSON.

6

Reactive update in Vue

Raw data lands in refs (allOrders, inventoryItems); computed properties derive totals, charts, and tables that re-render automatically.
